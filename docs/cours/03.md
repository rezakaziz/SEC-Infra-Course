# Séance 3 – Typologie des attaques logicielles et matérielles

## Objectifs de la séance

À l'issue de cette séance, l'étudiant sera capable de :

- Identifier les principales catégories d'attaques informatiques ;
- Comprendre les mécanismes de propagation des logiciels malveillants ;
- Reconnaître les impacts des attaques sur les systèmes d'information ;
- Comprendre les risques liés aux attaques matérielles ;
- Analyser des exemples réels d'attaques majeures.

---

# 1. Introduction aux cyberattaques (10 min)

## Pourquoi attaquer un système ?

Les motivations des attaquants peuvent être :

### Gain financier

Exemples :

- rançongiciels ;
- fraude ;
- vol de données.

---

### Espionnage

Exemples :

- informations stratégiques ;
- propriété intellectuelle ;
- données gouvernementales.

---

### Sabotage

Objectifs :

- perturber un service ;
- détruire des données ;
- interrompre une activité.

---

### Activisme

Actions visant à défendre une cause ou transmettre un message.

---

## Vecteurs d'attaque

Principaux moyens utilisés :

- courrier électronique ;
- téléchargement de logiciels ;
- vulnérabilités logicielles ;
- périphériques USB ;
- accès physiques.

---

# 2. Attaques logicielles (15 min)

## Virus

### Principe

Programme malveillant capable de s'attacher à un fichier ou à un programme.

### Caractéristiques

- nécessite généralement une action de l'utilisateur ;
- se propage en infectant d'autres fichiers.

### Conséquences

- corruption de données ;
- ralentissement du système ;
- compromission de l'intégrité.

---

## Vers (Worms)

### Principe

Programme capable de se propager automatiquement.

### Caractéristiques

- propagation rapide ;
- exploitation des réseaux.

### Conséquences

- saturation des ressources ;
- propagation massive.

---

## Chevaux de Troie (Trojans)

### Principe

Programme se présentant comme légitime mais contenant une fonctionnalité malveillante.

### Conséquences

- ouverture d'un accès distant ;
- vol d'informations ;
- prise de contrôle du système.

---

## Ransomwares

### Principe

Logiciel chiffrant les données de la victime.

### Objectif

Obtenir une rançon.

### Conséquences

- indisponibilité des données ;
- interruption de l'activité ;
- pertes financières.

---

# 3. Attaques système (10 min)

## Rootkits

### Principe

Logiciel permettant de dissimuler la présence d'un attaquant.

### Objectifs

- persistance ;
- dissimulation ;
- contrôle du système.

---

## Escalade de privilèges

### Principe

Obtention de privilèges supérieurs à ceux initialement accordés.

### Conséquences

- accès administrateur ;
- compromission complète du système.

---

## Exploitation de vulnérabilités

Exemples :

- dépassement de tampon ;
- injection de commandes ;
- vulnérabilités locales.

Objectif :

- exécuter du code ;
- accéder aux données ;
- prendre le contrôle du système.

---

# 4. Attaques matérielles (10 min)

## Vol d'équipement

Exemples :

- ordinateur portable ;
- disque dur ;
- support de sauvegarde.

### Risques

- fuite de données ;
- perte de disponibilité.

---

## Périphériques malveillants

Exemples :

- clé USB infectée ;
- périphérique se faisant passer pour un clavier.

### Conséquences

- infection du système ;
- exécution automatique d'actions.

---

## Keyloggers matériels

### Principe

Dispositif enregistrant les frappes clavier.

### Conséquences

- vol de mots de passe ;
- compromission de comptes.

---

# 5. Études de cas (10 min)

## WannaCry (2017)

### Type

Ransomware.

### Particularité

Propagation automatique via une vulnérabilité Windows.

### Impact

- centaines de milliers de systèmes affectés ;
- interruption de nombreux services.

---

## Stuxnet (2010)

### Type

Ver sophistiqué.

### Objectif

Sabotage d'infrastructures industrielles.

### Particularité

Première cyberarme largement médiatisée.

---

## NotPetya (2017)

### Type

Malware destructeur.

### Particularité

Présenté comme un ransomware mais conçu pour détruire les données.

### Impact

- entreprises internationales ;
- pertes financières majeures.

---

# 6. Conclusion (5 min)

## À retenir

- Les attaques peuvent être logicielles ou matérielles.
- Les logiciels malveillants prennent plusieurs formes :
  - virus ;
  - vers ;
  - chevaux de Troie ;
  - ransomwares.

- Les vulnérabilités permettent souvent l'escalade de privilèges et la compromission des systèmes.
- Les attaques matérielles restent une menace importante pour les données et les équipements.
- Comprendre les mécanismes d'attaque est indispensable pour mettre en œuvre des mesures de protection efficaces.

## Préparation du TP

Le TP portera sur :

- l'analyse d'un incident de sécurité ;
- l'identification de comportements suspects ;
- l'analyse de processus et de fichiers ;
- la détermination du type d'attaque et de ses conséquences.
