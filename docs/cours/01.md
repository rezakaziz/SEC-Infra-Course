# Séance 1 — Protection des données et sécurité de l'information dans les systèmes d'exploitation

## Objectifs de la séance

À l'issue de cette séance, l'étudiant sera capable de :

- Comprendre les objectifs du module et son organisation ;
- Identifier les principales données sensibles présentes sur un système Linux ;
- Comprendre les principes de confidentialité, d'intégrité et de disponibilité ;
- Identifier les mécanismes de protection proposés par un système d'exploitation ;
- Comprendre le rôle des utilisateurs, des groupes et des permissions dans la protection des données.

---

# 2. Introduction à la sécurité de l'information (10 min)

## Pourquoi protéger l'information ?

L'information constitue une ressource essentielle pour :

- les entreprises ;
- les administrations ;
- les particuliers.

Une compromission peut entraîner :

- une perte financière ;
- une interruption d'activité ;
- une atteinte à l'image ;
- des sanctions réglementaires.

## Menaces et risques

### Menace

Cause potentielle d'un incident.

Exemples :

- attaquant ;
- malware ;
- erreur humaine.

### Risque

Conséquence possible d'une menace exploitant une vulnérabilité.

# 3. Rappels Linux utiles à la sécurité (10 min)

## Utilisateurs et groupes

Linux est un système multi-utilisateurs.

Chaque action est réalisée au nom d'un utilisateur.

Notions :

- utilisateur ;
- groupe ;
- administrateur (root).

Exemples :

```bash
whoami
id
```

## Fichiers sensibles

Exemples :

```text
/etc/passwd
/etc/shadow
/home
/var/log
/var/www
```

## Permissions Linux

Principe :

```text
propriétaire
groupe
autres
```

Exemple :

```bash
ls -l
```

## Services système

Exemples :

- SSH
- Apache

Ces services seront utilisés tout au long du module.

# 4. Les principes fondamentaux de la sécurité (15 min)

## Confidentialité

Objectif :

Empêcher l'accès non autorisé aux données.

Exemples :

- mots de passe ;
- données RH ;
- données clients.

## Intégrité

Objectif :

Garantir que les données n'ont pas été modifiées de manière non autorisée.

Exemples :

- fichier de configuration ;
- base de données ;
- document administratif.

## Disponibilité

Objectif :

Garantir l'accès aux ressources lorsque nécessaire.

Exemples :

- serveur Web ;
- serveur de fichiers ;
- système d'information.
