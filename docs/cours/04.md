# Séance 4 – Authentification sous Linux : PAM et politiques de sécurité

## Objectifs de la séance

À l'issue de cette séance, l'étudiant sera capable de :

- Comprendre les mécanismes d'authentification sous Linux ;
- Distinguer identification, authentification et autorisation ;
- Comprendre le rôle de PAM ;
- Mettre en place des politiques de mots de passe ;
- Identifier les bonnes pratiques de sécurisation des comptes.

---

# 1. Introduction

## Pourquoi authentifier les utilisateurs ?

- Protection des données
- Protection des services
- Traçabilité des actions

### Exemple

Accès :

- au système Linux ;
- à un service SSH ;
- à une application Web.

---

# 2. Identification, authentification et autorisation

## Identification

Qui suis-je ?

Exemple :

```text
alice
```

---

## Authentification

Comment prouver mon identité ?

Exemples :

- mot de passe ;
- certificat ;
- second facteur.

---

## Autorisation

À quoi ai-je accès ?

Exemples :

- fichiers ;
- répertoires ;
- services.

---

# 3. Présentation de PAM

## Qu'est-ce que PAM ?

Pluggable Authentication Modules

Couche permettant de gérer l'authentification sous Linux.

---

## Principe de fonctionnement

```text
Application
      ↓
     PAM
      ↓
Modules
```

Exemples :

- login
- ssh
- sudo

---

## Architecture générale

- modules
- politiques
- fichiers de configuration

---

# 4. Gestion des mots de passe

## Complexité

Pourquoi imposer :

- longueur minimale ;

---

## Expiration

- durée de validité ;

---

## Historique

Empêcher la réutilisation immédiate des mots de passe.

# 5. Sécurisation des comptes

## Limitation des tentatives

Protection contre :

- attaques par dictionnaire ;
- force brute.

---

## Verrouillage des comptes

Blocage temporaire après plusieurs échecs.

---

## Comptes privilégiés

# 6. Bonnes pratiques

## Principe du moindre privilège

Donner uniquement les droits nécessaires.

---

## Authentification multifacteur (MFA)

Présentation générale :

- quelque chose que l'on connaît ;
- quelque chose que l'on possède ;
- quelque chose que l'on est.

---

# 7. Synthèse

## À retenir

- PAM centralise l'authentification sous Linux ;
- les mots de passe doivent être protégés par des politiques adaptées ;
- les comptes privilégiés doivent être utilisés avec précaution ;
- l'authentification constitue la première ligne de défense d'un système.
