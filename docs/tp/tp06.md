# Détection des attaques et investigation système avec journald et auditd

[⬇️ Télécharger la machine virtuelle (vm-sec-tp6.ova)](vm-sec-tp6.ova)

!!! note "Contexte"

    Un serveur Linux de l'entreprise a été compromis.

    L'équipe sécurité a réussi à contenir l'incident mais souhaite désormais comprendre ce qui s'est produit :

    - Comment l'attaquant est-il entré ?
    - Quels comptes ont été utilisés ?
    - Quelles actions ont été réalisées ?
    - Quels fichiers ont été modifiés ?

    Votre mission consiste à mener une investigation à partir des journaux système et des mécanismes d'audit du système d'exploitation.

---

## Exercice 1 : Découverte des journaux système

Avant de commencer l'investigation, il est nécessaire d'identifier les principales sources d'information.

!!! question "Question 1"

    Affichez les derniers événements enregistrés :

    ~~~bash
    journalctl -n 50
    ~~~

    Quels types d'événements observez-vous ?

!!! question "Question 2"

    Recherchez les journaux liés à l'authentification :

    ~~~bash
    journalctl -u ssh
    ~~~

    ou

    ~~~bash
    cat /var/log/auth.log
    ~~~

    Quels événements sont enregistrés ?

!!! question "Question 3"

    Pourquoi les journaux constituent-ils un élément essentiel dans la détection des incidents de sécurité ?

---

## Exercice 2 : Analyse des connexions

L'équipe sécurité suspecte une compromission de compte.

!!! question "Question 1"

    Recherchez les connexions réussies :

    ~~~bash
    grep "Accepted" /var/log/auth.log
    ~~~

    ou

    ~~~bash
    journalctl | grep "Accepted"
    ~~~

!!! question "Question 2"

    Recherchez les échecs d'authentification :

    ~~~bash
    grep "Failed" /var/log/auth.log
    ~~~

!!! question "Question 3"

    Identifiez :

    - le compte concerné ;
    - l'adresse IP impliquée ;
    - la période des événements.

## Exercice 3 : Reconstitution de la chronologie

L'objectif est maintenant de comprendre la succession des événements.

!!! question "Question 1"

    Recherchez les commandes exécutées après la connexion suspecte.

!!! question "Question 2"

    Identifiez les éventuels changements de privilèges :

    ~~~bash
    grep sudo /var/log/auth.log
    ~~~

    ou

    ~~~bash

    ~~~

!!! question "Question 3"

## Exercice 4 : Introduction à auditd

Le système utilise également le mécanisme d'audit Linux.

!!! question "Question 1"

    Vérifiez que le service est actif :

    ~~~bash
    systemctl status auditd
    ~~~

!!! question "Question 2"

    Affichez les règles d'audit configurées :

    ~~~bash
    sudo auditctl -l
    ~~~

    Quels fichiers ou événements sont surveillés ?

## Exercice 5 : Analyse des événements d'audit

L'entreprise souhaite savoir si des fichiers sensibles ont été consultés ou modifiés.

!!! question "Question 1"

    Recherchez les événements récents :

    ~~~bash
    sudo ausearch -ts today
    ~~~

!!! question "Question 2"

    Analysez les accès au fichier sensible surveillé :

    ~~~bash
    sudo ausearch -f /srv/rh/salaires.txt
    ~~~

!!! question "Question 3"

    Déterminez :

    - quel utilisateur a accédé au fichier ;
    - à quelle date ;
    - quelle opération a été réalisée.
