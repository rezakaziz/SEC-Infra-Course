# Authentification sous Linux avec PAM

[⬇️ Télécharger la machine virtuelle (vm-sec-tp3.ova)](vm-sec-tp3.ova)

!!! note "Contexte"

    Une entreprise souhaite renforcer la sécurité des accès à ses serveurs Linux.

    Plusieurs incidents ont été signalés :

    - utilisation de mots de passe faibles ;
    - tentatives répétées de connexion ;
    - absence de politique de renouvellement des mots de passe.

    Votre mission consiste à analyser la configuration actuelle du système et à mettre en œuvre des politiques d'authentification plus strictes à l'aide de PAM (*Pluggable Authentication Modules*).

---

## Exercice 1 : Découverte de PAM

Avant de modifier la configuration du système, il est nécessaire de comprendre le rôle de PAM.

!!! question "Question 1"

    Vérifiez la présence des fichiers de configuration PAM :

    ~~~bash
    ls /etc/pam.d
    ~~~

    À quoi semblent correspondre les différents fichiers présents ?

!!! question "Question 2"

    Examinez le contenu du fichier suivant :

    ~~~bash
    cat /etc/pam.d/common-auth
    ~~~

    Quels modules d'authentification sont utilisés ?

!!! question "Question 3"

    Recherchez la signification des mots-clés suivants :

    - required
    - requisite
    - sufficient
    - optional

    Quel est leur rôle dans la pile PAM ?

## Exercice 2 : Analyse de la politique actuelle

!!! question "Question 1"

    - la durée minimale ;
    - la période d'avertissement.

    ~~~

    Justifiez votre réponse.

## Exercice 3 : Renforcement des mots de passe

L'entreprise souhaite imposer des mots de passe plus robustes.

!!! question "Question 1"

    Identifiez le module PAM responsable du contrôle de complexité des mots de passe.

!!! question "Question 2"

    Configurez une politique imposant :

    - une longueur minimale de 10 caractères ;
    - au moins une majuscule ;
    - au moins une minuscule ;
    - au moins un chiffre.

!!! question "Question 3"

    Testez la nouvelle politique en essayant de définir :

    - un mot de passe faible ;
    - un mot de passe conforme.

    Que constatez-vous ?

## Exercice 4 : Limitation des tentatives de connexion

Afin de limiter les attaques par force brute, l'entreprise souhaite bloquer temporairement les comptes après plusieurs échecs.

!!! question "Question 1"

    Identifiez le module PAM permettant de limiter les tentatives d'authentification.

!!! question "Question 2"

    Configurez le système afin :

    - d'autoriser au maximum 3 tentatives ;
    - de bloquer l'utilisateur pendant 60 secondes après dépassement.

!!! question "Question 3"

    Testez cette politique en saisissant plusieurs mots de passe incorrects.

    Quel comportement observez-vous ?
