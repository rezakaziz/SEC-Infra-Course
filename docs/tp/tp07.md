# Durcissement d'un serveur Linux

[⬇️ Télécharger la machine virtuelle (vm-sec-tp7.ova)](vm-sec-tp7.ova)

!!! note "Contexte"

    Une PME souhaite renforcer la sécurité de son serveur Linux hébergeant plusieurs services :

    - SSH ;
    - Apache ;
    - services système.

    Un audit préliminaire a révélé plusieurs faiblesses :

    - comptes mal configurés ;
    - services inutiles actifs ;
    - authentification SSH insuffisamment sécurisée ;
    - absence de politique de durcissement.

    Votre mission consiste à réduire la surface d'attaque du système en appliquant plusieurs mesures de sécurité.

---

## Exercice 1 : Audit initial du système

Avant toute modification, il est nécessaire d'évaluer l'état de sécurité du serveur.

!!! question "Question 1"

    Identifiez les services actifs :

    ~~~bash
    systemctl list-units --type=service --state=running
    ~~~

    Quels services semblent indispensables ?

!!! question "Question 2"

    Identifiez les ports ouverts :

    ~~~bash
    ss -tulpn
    ~~~

    Quels services sont accessibles depuis le réseau ?

!!! question "Question 3"

    Quels risques peuvent être associés à l'exposition de services inutiles ?

---

## Exercice 2 : Réduction de la surface d'attaque

L'objectif est de limiter le nombre de services exposés.

!!! question "Question 1"

    Identifiez les services inutiles présents sur le serveur.

!!! question "Question 2"

    Désactivez un service non nécessaire :

    ~~~bash
    sudo systemctl disable nom_service
    sudo systemctl stop nom_service
    ~~~

!!! question "Question 3"

    Vérifiez que le service n'est plus accessible.

---

## Exercice 3 : Sécurisation du service SSH

Le serveur autorise actuellement les connexions administrateur directes.

!!! question "Question 1"

    Analysez la configuration SSH :

    ~~~bash
    sudo cat /etc/ssh/sshd_config
    ~~~

    Relevez les paramètres liés à :

    - l'authentification ;
    - l'utilisateur root ;
    - les tentatives de connexion.

!!! question "Question 2"

    Modifiez la configuration afin :

    - d'interdire la connexion directe de root ;
    - de limiter les tentatives de connexion ;
    - de réduire les risques d'attaque par force brute.

!!! question "Question 3"

    Redémarrez le service SSH :

    ~~~bash
    sudo systemctl restart ssh
    ~~~

    Vérifiez que la configuration est prise en compte.

## Exercice 4 : Sécurisation des comptes utilisateurs

L'entreprise souhaite appliquer le principe du moindre privilège.

!!! question "Question 1"

    Listez les utilisateurs disposant de privilèges administrateur.

!!! question "Question 2"

    Vérifiez les appartenances aux groupes :

    ~~~bash
    groups
    ~~~

!!! question "Question 3"

    Identifiez les comptes présentant un risque de sécurité.

!!! question "Question 4"

    Proposez des mesures permettant de réduire ces risques.

## Exercice 5 : Sécurisation du serveur Web

Le serveur Apache expose plusieurs informations techniques.

!!! question "Question 1"

    Identifiez la version du serveur :

    ~~~bash
    curl -I http://localhost
    ~~~

!!! question "Question 2"

    Analysez la configuration Apache.

!!! question "Question 3"

    Désactivez l'affichage des informations inutiles :

    - version Apache ;
    - détails du système ;
    - informations de débogage.

!!! question "Question 4"

    Vérifiez les modifications réalisées.

## Exercice 6 : Audit de sécurité avec Lynis

L'entreprise souhaite réaliser un audit automatisé.

!!! question "Question 1"

    Lancez l'outil Lynis :

    ~~~bash
    sudo lynis audit system
    ~~~

!!! question "Question 2"

    Relevez :

    - le score obtenu ;
    - les principales recommandations.

!!! question "Question 3"

    Choisissez trois recommandations et appliquez-les.
