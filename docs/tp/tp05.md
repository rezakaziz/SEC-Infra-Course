# Analyse d'un incident de sécurité : malware, rootkit et ransomware

[⬇️ Télécharger la machine virtuelle (vm-sec-tp5.ova)](vm-sec-tp5.ova)

!!! note "Contexte"

    Une PME a constaté plusieurs comportements anormaux sur un serveur Linux :

    - ralentissements inhabituels ;
    - connexions suspectes ;
    - création de fichiers inconnus ;
    - disparition de certains documents.

    L'équipe sécurité suspecte la présence d'un logiciel malveillant.

    Votre mission consiste à analyser les traces laissées sur le système afin d'identifier l'attaque, comprendre son fonctionnement et évaluer son impact.

---

## Exercice 1 : Observation du système

Avant de rechercher l'origine de l'incident, il est nécessaire d'observer l'état du système.

!!! question "Question 1"

    Affichez les processus actifs :

    ~~~bash
    ps aux
    ~~~

    Relevez les processus qui vous semblent inhabituels.

!!! question "Question 2"

    Examinez les connexions réseau actives :

    ~~~bash
    ss -tulpn
    ~~~

    ou

    ~~~bash
    netstat -tulpn
    ~~~

    Identifiez les services à l'écoute et les connexions suspectes.

!!! question "Question 3"

    Analysez les tâches planifiées :

    ~~~bash
    crontab -l
    sudo ls /etc/cron.*
    ~~~

    Un mécanisme de persistance semble-t-il présent ?

---

## Exercice 2 : Recherche d'indicateurs de compromission

L'entreprise suspecte l'exécution d'un programme malveillant.

!!! question "Question 1"

    Recherchez les fichiers récemment modifiés :

    ~~~bash
    find /home -type f -mtime -7
    ~~~

    Quels fichiers attirent votre attention ?

!!! question "Question 2"

    Recherchez des exécutables inhabituels :

    ~~~bash
    find /tmp /var/tmp -type f -executable
    ~~~

    Pourquoi ces répertoires sont-ils souvent utilisés par les attaquants ?

!!! question "Question 3"

    Analysez les répertoires suivants :

    ~~~bash
    /tmp
    /var/tmp
    /dev/shm
    ~~~

    Quels éléments semblent suspects ?

## Exercice 3 : Analyse des journaux système

Les journaux constituent une source d'information essentielle lors d'une investigation.

!!! question "Question 1"

    Consultez les événements récents :

    ~~~bash
    journalctl -xe
    ~~~

!!! question "Question 2"

    Analysez les tentatives de connexion :

    ~~~bash
    grep "Failed" /var/log/auth.log
    ~~~

    ou

    ~~~bash
    journalctl -u ssh
    ~~~

!!! question "Question 3"

    Reconstituez la chronologie des événements ayant conduit à l'incident.
