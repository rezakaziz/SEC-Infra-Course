# Protection des données et sécurisation d'un serveur Web

[⬇️ Télécharger la machine virtuelle (vm-sec-tp1.ova)](vm-sec-tp1.ova)

!!! note "Contexte"

    Une machine virtuelle Linux vous est fournie avec un serveur Apache déjà installé et fonctionnel.

    Le serveur héberge une application Web interne contenant des informations sensibles. Les communications entre les clients et le serveur ne sont actuellement pas protégées.

    L'objectif de ce TP est d'analyser les risques liés à cette situation puis de mettre en œuvre une solution permettant de protéger les données échangées.

---

## Exercice 1 : Identification des données sensibles

Avant de sécuriser un système, il est nécessaire d'identifier les données qu'il héberge et les risques associés.

!!! question "Question 1"

    Examinez les éléments suivants :

    ~~~bash
    ls -l /etc/passwd
    ls -l /etc/shadow
    ls -l /var/www/html
    ls -l /home
    ~~~

    Pour chacun d'eux, indiquez :

    - son rôle ;
    - son niveau de sensibilité ;
    - les conséquences d'une divulgation non autorisée.

!!! question "Question 2"

    Donnez une définition des propriétés suivantes :

    - Confidentialité ;
    - Intégrité ;
    - Disponibilité.

    Illustrez chacune d'elles à l'aide d'un exemple observé sur la machine virtuelle.

## Exercice 3 : Sécurisation du serveur Apache

Nous allons maintenant protéger les échanges entre le client et le serveur.

!!! question "Question 1"

    Générez une clé privée et un certificat auto-signé :

    ~~~bash
    sudo openssl req -x509 \
      -newkey rsa:2048 \
      -nodes \
      -days 365 \
      -keyout server.key \
      -out server.crt
    ~~~

!!! question "Question 2"

    Activez le module SSL d'Apache :

    ~~~bash
    sudo a2enmod ssl
    ~~~

!!! question "Question 3"

    Activez la configuration HTTPS :

    ~~~bash
    sudo a2ensite default-ssl
    ~~~

!!! question "Question 4"

    Configurez Apache afin d'utiliser le certificat et la clé précédemment générés.

    Vérifiez ensuite la configuration :

    ~~~bash
    sudo apachectl configtest
    ~~~

!!! question "Question 5"

    Redémarrez le serveur Apache :

    ~~~bash
    sudo systemctl restart apache2
    ~~~

    Vérifiez son bon fonctionnement.
