    Séance 2 – Vulnérabilités des systèmes d'exploitation

    Objectifs de la séance

    À l'issue de cette séance, l'étudiant sera capable de :

    • Comprendre la notion de vulnérabilité ;
    • Identifier les principales vulnérabilités des systèmes Linux ;
    • Comprendre les conséquences de leur exploitation ;
    • Reconnaître les situations favorisant une élévation de privilèges ;
    • Connaître les principes de gestion des vulnérabilités.

    1. Introduction aux vulnérabilités (10 min)

    Définition

    Une vulnérabilité est une faiblesse pouvant être exploitée pour compromettre la sécurité d'un système.

    Une vulnérabilité peut affecter :

    • le système d'exploitation ;
    • une application ;
    • un service ;
    • une configuration.

    Surface d'attaque

    La surface d'attaque correspond à l'ensemble des points d'entrée pouvant être ciblés par un attaquant.

    Exemples :

    • services réseau ;
    • applications Web ;
    • comptes utilisateurs ;
    • fichiers accessibles.

    Cycle de vie d'une vulnérabilité

    Découverte
      ↓
Publication
      ↓
Exploitation
      ↓
Correctif
      ↓
Mise à jour

    2. Erreurs de configuration (15 min)

    Permissions excessives

    Exemples :

    • répertoire accessible à tous ;
    • fichier sensible en lecture publique ;
    • droits d'écriture inutiles.

    Conséquences :

    • fuite de données ;
    • modification non autorisée.

    Services mal configurés

    Exemples :

    • service exposé inutilement ;
    • configuration par défaut conservée ;
    • fonctionnalités inutiles activées.

    Conséquences :

    • augmentation de la surface d'attaque ;
    • compromission du service.

    Comptes privilégiés

    Exemples :

    • utilisation abusive de root ;
    • comptes administrateurs partagés ;
    • mots de passe faibles.

    Conséquences :

    • accès complet au système ;
    • difficulté de traçabilité.

    3. Vulnérabilités logicielles (15 min)

    Dépassement de tampon (Buffer Overflow)

    Principe

    Un programme reçoit plus de données qu'il ne peut en stocker.

    Conséquences

    • plantage ;
    • exécution de code malveillant ;
    • prise de contrôle du programme.

    Injection de commandes

    Principe

    Une application exécute une commande système construite à partir d'une entrée utilisateur.

    Conséquences

    • exécution de commandes non autorisées ;
    • accès aux fichiers ;
    • compromission du système.

    Traversée de répertoires

    Principe

    Accéder à des fichiers situés en dehors du répertoire prévu.

    Exemple :

    ../../etc/passwd

    Conséquences

    • divulgation d'informations ;
    • découverte de la configuration du système.

    4. Élévation de privilèges (10 min)

    Privilèges utilisateurs et administrateurs

    Sous Linux :

    • utilisateur standard ;
    • administrateur (root).

    Vulnérabilités locales

    Un utilisateur obtient davantage de privilèges que prévu.

    Exemples :

    • programme SUID vulnérable ;
    • permissions incorrectes ;
    • mauvaise configuration système.

    Mauvaises pratiques

    • exécution systématique en root ;
    • permissions excessives ;
    • absence de séparation des rôles.

    5. Gestion des vulnérabilités (5 min)

    Identification

    • veille de sécurité ;
    • audit ;
    • inventaire des systèmes.

    Correction

    • modification des configurations ;
    • suppression des services inutiles ;
    • correction des permissions.

    Mise à jour

    Importance :

    • correction des failles connues ;
    • réduction des risques ;
    • maintien du niveau de sécurité.

    6. Conclusion (5 min)

    Réduction de la surface d'attaque

    Bonnes pratiques :

    • désactiver les services inutiles ;
    • limiter les privilèges ;
    • appliquer les mises à jour ;
    • contrôler les permissions ;
    • surveiller les vulnérabilités publiées.

    À retenir

    • Une vulnérabilité est une faiblesse exploitable.
    • Les erreurs de configuration sont une cause fréquente d'incidents.
    • Les vulnérabilités logicielles peuvent compromettre un système entier.
    • L'élévation de privilèges représente un risque majeur.
    • La gestion proactive des vulnérabilités est un élément essentiel de la sécurité des systèmes.
