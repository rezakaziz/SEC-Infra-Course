    Séance 1 — Protection des données et sécurité de l'information dans les systèmes d'exploitation

    Objectifs de la séance

    À l'issue de cette séance, l'étudiant sera capable de :

    • Comprendre les objectifs du module et son organisation ;
    • Identifier les principales données sensibles présentes sur un système Linux ;
    • Comprendre les principes de confidentialité, d'intégrité et de disponibilité ;
    • Identifier les mécanismes de protection proposés par un système d'exploitation ;
    • Comprendre le rôle des utilisateurs, des groupes et des permissions dans la protection des données.

    1. Présentation du module (10 min)

    Objectifs du module

    Le module vise à comprendre les mécanismes permettant de sécuriser :

    • les systèmes d'exploitation ;
    • les données ;
    • les services ;
    • les plateformes logicielles.

    Organisation du module

    Thèmes abordés

    • Protection des données
    • Vulnérabilités des systèmes
    • Attaques informatiques
    • Authentification
    • Contrôle d'accès
    • Confinement des applications
    • Défense en profondeur
    • Vulnérabilités applicatives

    Déroulement pédagogique

    Chaque séance comprend :

    • 1 heure de cours ;
    • 2 heures de travaux pratiques.

    Les travaux pratiques sont réalisés sur une machine virtuelle Debian utilisée pendant tout le module.

    Évaluation

    • Quiz de 15 minutes à chaque séance ;
    • Travaux pratiques ;
    • Contrôle continu.

    2. Introduction à la sécurité de l'information (10 min)

    Pourquoi protéger l'information ?

    L'information constitue une ressource essentielle pour :

    • les entreprises ;
    • les administrations ;
    • les particuliers.

    Une compromission peut entraîner :

    • une perte financière ;
    • une interruption d'activité ;
    • une atteinte à l'image ;
    • des sanctions réglementaires.

    Menaces et risques

    Menace

    Cause potentielle d'un incident.

    Exemples :

    • attaquant ;
    • malware ;
    • erreur humaine.

    Risque

    Conséquence possible d'une menace exploitant une vulnérabilité.

    3. Rappels Linux utiles à la sécurité (10 min)

    Utilisateurs et groupes

    Linux est un système multi-utilisateurs.

    Chaque action est réalisée au nom d'un utilisateur.

    Notions :

    • utilisateur ;
    • groupe ;
    • administrateur (root).

    Exemples :

    whoami
id

    Fichiers sensibles

    Exemples :

    /etc/passwd
/etc/shadow
/home
/var/log
/var/www

    Permissions Linux

    Principe :

    propriétaire
groupe
autres

    Exemple :

    ls -l

    Services système

    Exemples :

    • SSH
    • Apache

    Ces services seront utilisés tout au long du module.

    4. Les principes fondamentaux de la sécurité (15 min)

    Confidentialité

    Objectif :

    Empêcher l'accès non autorisé aux données.

    Exemples :

    • mots de passe ;
    • données RH ;
    • données clients.

    Intégrité

    Objectif :

    Garantir que les données n'ont pas été modifiées de manière non autorisée.

    Exemples :

    • fichier de configuration ;
    • base de données ;
    • document administratif.

    Disponibilité

    Objectif :

    Garantir l'accès aux ressources lorsque nécessaire.

    Exemples :

    • serveur Web ;
    • serveur de fichiers ;
    • système d'information.

    5. Les données sensibles dans un système Linux (10 min)

    Données utilisateurs

    /home

    Données système

    /etc/passwd
/etc/shadow

    Données applicatives

    /var/www
/var/lib

    Journaux

    /var/log

    Pourquoi ces données sont-elles sensibles ?

    Analyse des impacts sur :

    • la confidentialité ;
    • l'intégrité ;
    • la disponibilité.

    6. Les mécanismes de protection du système d'exploitation (5 min)

    Présentation des mécanismes qui seront étudiés durant le module :

    • authentification ;
    • permissions ;
    • ACL ;
    • PAM ;
    • AppArmor ;
    • défense en profondeur.

    7. Synthèse et préparation du TP (5 min)

    À retenir

    • Les données constituent un actif à protéger.
    • La sécurité repose sur la confidentialité, l'intégrité et la disponibilité.
    • Linux protège les ressources grâce aux utilisateurs, groupes et permissions.
    • Les mécanismes de sécurité étudiés dans le module visent à protéger les données et les services du système.

    Préparation du TP

    Le TP portera sur :

    • l'identification des données sensibles ;
    • l'analyse des permissions ;
    • la protection des données dans un système Linux.
