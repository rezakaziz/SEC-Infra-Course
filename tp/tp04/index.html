    TP — Attaques Server-Side avec DVWA

    Contexte

    Les applications Web modernes reposent sur une architecture composée d'un navigateur, d'un serveur applicatif et d'une base de données.

    Comme vu en cours, les vulnérabilités côté serveur (Server-Side) apparaissent lorsque les données fournies par l'utilisateur sont utilisées sans validation suffisante ou lorsque les contrôles de sécurité sont mal implémentés.

    Dans ce TP, vous utiliserez l'application DVWA (Damn Vulnerable Web Application) afin d'observer et d'analyser plusieurs vulnérabilités étudiées en cours :

    • SQL Injection ;
    • Command Injection ;
    • File Inclusion ;
    • File Upload.

    L'objectif n'est pas d'apprendre à attaquer des systèmes réels mais de comprendre les mécanismes exploités par les attaquants afin de mieux les sécuriser.

    Objectifs

    À l'issue de ce TP, vous serez capable de :

    • Identifier une vulnérabilité SQL Injection ;
    • Comprendre le fonctionnement d'une Command Injection ;
    • Analyser une vulnérabilité de File Inclusion ;
    • Évaluer les risques liés aux fonctionnalités d'upload ;
    • Identifier les composants serveur ciblés ;
    • Comprendre les impacts potentiels sur les données et l'infrastructure ;
    • Proposer des contre-mesures adaptées.

    Important

    Toutes les manipulations doivent être réalisées uniquement sur l'environnement de TP fourni.

    Les vulnérabilités observées dans DVWA sont volontairement présentes afin de faciliter l'apprentissage.

    Toute utilisation en dehors du cadre pédagogique est interdite.

    À retenir

    • Les attaques Server-Side ciblent les composants exécutés sur le serveur.
    • Les SQL Injections ciblent les bases de données.
    • Les Command Injections ciblent le système d'exploitation.
    • Les vulnérabilités de File Inclusion permettent l'accès à des fichiers non prévus.
    • Les fonctionnalités d'upload doivent considérer tout fichier comme potentiellement malveillant.
    • Le principe fondamental reste : ne jamais faire confiance aux entrées utilisateur.
