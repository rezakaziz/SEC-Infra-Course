    Vulnérabilités des systèmes d’exploitation : élévation de privilèges

    ⬇️ Télécharger la machine virtuelle (vm-sec-tp2.ova)

    Contexte

    Une entreprise met à disposition de ses employés une application interne permettant de consulter leur dossier RH.

    Cette application s'exécute avec des privilèges élevés afin d'accéder aux données stockées dans un répertoire protégé.

    Suite à plusieurs signalements, l'équipe sécurité suspecte qu'un utilisateur puisse accéder à des informations qui ne lui sont pas destinées.

    Votre mission consiste à analyser le fonctionnement de l'application, identifier la vulnérabilité présente et proposer les mesures correctives appropriées.

    Exercice 1 : Découverte de l'environnement

    Avant d'analyser la vulnérabilité, il est nécessaire de comprendre l'environnement dans lequel l'application s'exécute.

    Question 1

    Connectez-vous avec le compte utilisateur fourni :

    id
whoami

    Quel est votre niveau de privilège ?

    Question 2

    Examinez les permissions du programme :

    ls -l /usr/local/bin/consultation

    Quel est le propriétaire du programme ?

    Question 3

    Recherchez les programmes disposant du bit SUID :

    find / -perm -4000 -type f 2>/dev/null

    Quel est le rôle du mécanisme SUID ?

    Exercice 2 : Utilisation normale de l'application

    L'application permet normalement de consulter les dossiers RH des employés.

    Question 1

    Exécutez la commande suivante :

    consultation employe1

    Quel est le résultat obtenu ?

    Question 2

    Consultez plusieurs dossiers disponibles.

    Quel semble être le fonctionnement de l'application ?

    Question 3

    Déterminez dans quel répertoire les informations sont stockées.

    Exercice 3 : Analyse de la vulnérabilité

    L'application construit dynamiquement le chemin du fichier à consulter à partir du paramètre fourni par l'utilisateur.

    Question 1

    Testez différents paramètres afin de comprendre comment le programme construit le chemin d'accès aux fichiers.

    Question 2

    Essayez d'accéder à un fichier situé en dehors du répertoire RH.

    Par exemple :

    consultation ../../../etc/passwd

    Le programme empêche-t-il cet accès ?

    Question 3

    Identifiez la vulnérabilité exploitée.

    Indiquez :

    • son nom ;
    • son principe ;
    • ses conséquences potentielles.

    Exercice 4 : Exploitation de la vulnérabilité

    Le répertoire suivant contient des documents confidentiels :

    /home/admin/confidentiel/

    Question 1

    Utilisez la vulnérabilité identifiée afin d'accéder à un document appartenant à l'administrateur.

    Question 2

    Pourquoi cet accès n'est-il normalement pas autorisé ?

    Question 3

    Expliquez pourquoi cette situation constitue une élévation de privilèges.

    Exercice 6 : Remédiation

    L'équipe sécurité souhaite corriger la vulnérabilité.

    Question 1

    Proposez une méthode permettant de limiter l'accès uniquement aux dossiers RH autorisés.

    Question 2

    Le programme a-t-il réellement besoin de privilèges élevés ?

    Justifiez votre réponse.

    Question 3

    Quelles mesures complémentaires pourraient être mises en œuvre ?

    Parmi les mécanismes étudiés ou à venir :

    • PAM ;
    • ACL ;
    • SELinux ;
    • AppArmor ;
    • journalisation.

    Exercice 7 : Synthèse

    Question

    Rédigez une courte synthèse répondant aux questions suivantes :

    • Quelle vulnérabilité a été exploitée ?
    • Comment a-t-elle permis un accès non autorisé ?
    • Quels risques présente-t-elle pour l'entreprise ?
    • Quelles mesures permettraient d'éviter ce type d'incident ?

    Note

    Cette séance a illustré les conséquences d'une mauvaise gestion des privilèges dans un système d'exploitation.

    Dans la prochaine séance, nous étudierons les mécanismes d'authentification sous Linux et la manière dont PAM permet de renforcer la sécurité des accès aux ressources du système.
