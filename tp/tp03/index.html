    TP — Découverte des vulnérabilités Web avec DVWA

    Contexte

    Les applications Web modernes reposent sur un navigateur, un serveur Web et des mécanismes d'authentification permettant de maintenir une session utilisateur.

    Comme vu en cours, plusieurs vulnérabilités peuvent apparaître lorsque les données fournies par l'utilisateur ne sont pas correctement contrôlées.

    Dans ce TP, vous utiliserez l'application DVWA (Damn Vulnerable Web Application) afin d'observer et d'analyser plusieurs vulnérabilités étudiées en cours :

    • sessions Web ;
    • cookies ;
    • Reflected XSS ;
    • Stored XSS ;
    • DOM-Based XSS ;
    • CSRF.

    L'objectif n'est pas d'apprendre à attaquer des systèmes réels mais de comprendre les mécanismes exploités par les attaquants afin de mieux les sécuriser.

    Objectifs

    À l'issue de ce TP, vous serez capable de :

    • Vérifier l'accessibilité d'une application Web dans une machine virtuelle ;
    • Comprendre le mécanisme de redirection de ports VirtualBox ;
    • Identifier un cookie de session ;
    • Comprendre le fonctionnement d'une authentification Web ;
    • Observer le comportement du navigateur face à une vulnérabilité XSS ;
    • Différencier les trois grandes familles de XSS ;
    • Comprendre le principe d'une attaque CSRF ;
    • Identifier les impacts potentiels sur les utilisateurs ;
    • Proposer des contre-mesures adaptées.

    Important

    Toutes les manipulations doivent être réalisées uniquement sur l'environnement de TP fourni.

    Les vulnérabilités observées dans DVWA sont volontairement présentes afin de faciliter l'apprentissage.

    Toute utilisation en dehors du cadre pédagogique est interdite.

    Compte rendu

    Un compte rendu individuel est attendu à la fin de la séance.

    Pour chaque exercice, vous devrez fournir :

    • les manipulations réalisées ;
    • les captures d'écran demandées ;
    • les réponses aux questions ;
    • les conclusions de sécurité.

    Le compte rendu devra être remis à la fin de la séance.

    Exercice 5 : CSP Bypass

    Contexte

    La Content Security Policy (CSP) est un mécanisme de sécurité implémenté par les navigateurs afin de limiter l'exécution de contenus potentiellement dangereux.

    Une CSP permet notamment de contrôler :

    • les scripts JavaScript autorisés ;
    • les feuilles de style ;
    • les images ;
    • les ressources externes chargées par la page.

    Lorsqu'une politique CSP est mal configurée, il peut être possible de contourner certaines protections et d'exécuter du code malgré les restrictions prévues.

    Cet exercice vise à observer le fonctionnement de la CSP et à comprendre les conséquences d'une configuration insuffisante.

    À retenir

    • Une session Web repose généralement sur un cookie.
    • Le navigateur joue un rôle central dans la sécurité des applications Web.
    • Les vulnérabilités XSS permettent l'exécution de code dans le navigateur de la victime.
    • Les attaques CSRF exploitent la confiance accordée à une session authentifiée.
    • La validation des entrées, l'encodage des sorties et la protection des sessions sont essentiels pour sécuriser une application Web.
