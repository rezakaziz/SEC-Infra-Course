    Audit et sécurisation d'une plateforme Linux

    ⬇️ Télécharger la machine virtuelle (vm-sec-tp8.ova)

    Contexte

    Vous êtes consultant en cybersécurité.

    Une PME souhaite réaliser un audit de sécurité de son serveur Linux hébergeant une application Web interne.

    Plusieurs mécanismes de sécurité ont été négligés :

    • comptes utilisateurs mal configurés ;
    • permissions excessives ;
    • services inutiles actifs ;
    • absence de politique de sécurité ;
    • défauts de configuration du serveur Web.

    Votre mission consiste à identifier les vulnérabilités présentes puis à mettre en œuvre les mesures correctives appropriées.

    Ce TP mobilise l'ensemble des compétences acquises durant le module.

    Exercice 1 : Cartographie du système

    Avant toute intervention, il est nécessaire d'identifier les composants de la plateforme.

    Question 1

    Identifiez les utilisateurs présents sur le système :

    cat /etc/passwd

    Question 2

    Listez les services actifs :

    systemctl list-units --type=service --state=running

    Question 3

    Identifiez les ports ouverts :

    ss -tulpn

    Question 4

    Réalisez un schéma simplifié de la plateforme :

    • utilisateurs ;
    • services ;
    • données ;
    • applications.

    Exercice 2 : Audit des comptes utilisateurs

    L'entreprise souhaite vérifier le respect du principe du moindre privilège.

    Question 1

    Identifiez les comptes disposant de privilèges administrateur.

    Question 2

    Recherchez :

    • les comptes inactifs ;
    • les comptes inutilisés ;
    • les comptes présentant des privilèges excessifs.

    Question 3

    Proposez des mesures correctives.

    Exercice 3 : Audit des permissions

    L'application stocke des informations sensibles dans plusieurs répertoires.

    Question 1

    Analysez les permissions des répertoires suivants :

    ls -lR /srv

    Question 2

    Identifiez :

    • les fichiers accessibles à tous ;
    • les permissions excessives ;
    • les données sensibles insuffisamment protégées.

    Question 3

    Corrigez les permissions identifiées.

    Exercice 4 : Contrôle d'accès

    L'entreprise souhaite mettre en place une politique d'accès plus restrictive.

    Question 1

    Analysez les ACL existantes :

    getfacl -R /srv

    Question 2

    Vérifiez la configuration SELinux ou AppArmor.

    Question 3

    Identifiez les améliorations possibles.

    Exercice 5 : Audit des services

    Question 1

    Analysez la configuration du serveur SSH.

    Question 2

    Analysez la configuration du serveur Apache.

    Question 3

    Identifiez les services inutiles ou mal configurés.

    Question 4

    Appliquez les corrections nécessaires.

    Exercice 6 : Analyse des journaux

    Une activité suspecte a été signalée.

    Question 1

    Consultez les journaux système :

    journalctl

    Question 2

    Recherchez :

    • les échecs d'authentification ;
    • les changements de privilèges ;
    • les erreurs de sécurité.

    Question 3

    Déterminez si une compromission semble avoir eu lieu.

    Exercice 7 : Audit automatisé

    L'entreprise souhaite disposer d'une évaluation globale de la sécurité du serveur.

    Question 1

    Lancez un audit :

    sudo lynis audit system

    Question 2

    Relevez :

    • le score obtenu ;
    • les vulnérabilités identifiées ;
    • les recommandations proposées.

    Question 3

    Corrigez les faiblesses les plus critiques.

    Exercice 9 : Rapport d'audit

    Question

    Rédigez un rapport de sécurité contenant :

    1. État initial de la plateforme

    • services présents ;
    • comptes utilisateurs ;
    • niveau de sécurité observé.

    2. Vulnérabilités identifiées

    • description ;
    • impact ;
    • criticité.

    3. Mesures de sécurisation appliquées

    • contrôle d'accès ;
    • sécurisation des services ;
    • durcissement du système ;
    • amélioration de la supervision.

    4. État final de la plateforme

    • améliorations apportées ;
    • risques résiduels ;
    • recommandations complémentaires.

    Note

    Ce TP constitue une synthèse de l'ensemble du module :

    • protection des données ;
    • gestion des vulnérabilités ;
    • authentification ;
    • contrôle d'accès ;
    • SELinux et AppArmor ;
    • analyse des incidents ;
    • audit système ;
    • durcissement des infrastructures.

    L'objectif n'est plus d'appliquer un mécanisme particulier mais d'adopter une démarche globale d'analyse et de sécurisation d'une plateforme logicielle.
